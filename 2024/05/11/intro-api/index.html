什么是API ?

我们经常在网上看到：

其实，以我的理解来看，API就是各个程序之间的契约或者合约。

那么，究竟什么是程序之间的合约呢？

举个例子，到了周末你一个人在家，父母要你打扫整个屋子的卫生，你贼懒不想打扫卫生，你就想请一个阿姨帮忙打扫。这个时候假如你要去保姆公司请阿姨，那么阿姨就一定是来打扫卫生，而不是给你做饭，帮你写统计学作业的。作为一个用户，你一定是了解你找到的服务是什么样的。

于是，阿姨来到你家里，开始辛苦打扫卫生。打扫的过程中，阿姨会问你：小伙子，你家垃圾桶在哪里呀？作为一个正常人，听到阿姨的要求你自然就会告诉阿姨垃圾桶的位置。那么我们说，作为用户的你，给服务提供商提供了必要的信息。

家政公司提供了一个API，这个API的唯一作用就是帮你打扫卫生。而作为用户的你，为了让API完成它的工作，必须提供一些必要的信息，比如垃圾桶的位置。

这就是程序之间的契约。

API到底是函数，还是什么？

api形式有多种，比如出现在程序语言中，出现在安装的依赖中，出现在实际提供的服务上。其实api的形式在很多情况下确实是以一个函数的形式出现的

出现在程序中

下面展示的是调用谷歌翻译的API：

定义了一个名为 translate_text 的函数，它接受要翻译的文本和目标语言代码作为参数，并返回翻译后的文本。

调用OpenWeatherMap API的天气预报函数：

定义了一个 get_weather 函数，接受城市名称和 OpenWeatherMap API 密钥作为参数，并返回该城市的天气信息。

以上是出现在程序中。

出现在安装依赖中

这些库都用自己的API，实际也是全世界程序员编写完封装的上传的，给到需要的人直接调用即可。

出现在实际服务中

首先你是一个人，你的力量很小。

你不可能为了知道明天天气怎么样，你去自己建一个气象站。你不能为了给租婆付房租，就自己写一个程序付钱。

因此你就需要借助第三方已经做的服务，比如天气预报，微信支付，支付宝等，以此来完成你的需求，而不需要很强的能力。

API是谁提供的？

以网络请求为基础的API，通常是互联网公司自己编写和提供的，他们希望自己的服务可以被更广泛的人群使用，作为开发者的你，如果用了他们提供的API，那么某种程序上你可能帮助了他们传播他们的服务。

另一方面，有时候API会直接向开发者收费，比如gtp4的服务，你想要调用他们的API，你得先支付一定费用，然后才能给你api，并且这个api是有分层级的，在调用次数和调用时间方面有所限制。

结语

1000个读者有1000个哈姆雷特，以上只是我对于api的理解，其实我也有很多理解不到位的地方，欢迎大家指正一下啊。
